# schemas — frozen §3 data contracts (W0)

The single source of truth for the interfaces every stage communicates through.
**Do not redefine these shapes elsewhere; import from here.** Sub-agents may
import this package but must never modify it (CLAUDE.md §9).

## Source of truth
`python/asl_schemas/models.py` (pydantic v2). Everything else is derived:

| Artifact | Path | Generated by |
|---|---|---|
| Python models + validators | `python/asl_schemas/` | — (source) |
| Canonical JSON Schema | `json/*.schema.json` | `python -m asl_schemas.gen_json_schema` |
| TypeScript types (frontend) | `ts/src/index.ts` | hand-mirrored, kept in sync |
| Example clip (reference) | `json/examples/*.clip.json` | committed |

Contracts covered: `SMPLXFrame`, `SMPLXSequence` (§3.1), `SMPLXClip` (§3.2),
`GlossSequence` (§3.3), `VapiTranscriptWebhook` (§3.4), `SignRequest`/`SignError` (§3.5).

## Why this rejects malformed clips
Every model sets `extra="forbid"` and every pose vector is a fixed-length list
(`body_pose` 63, hands 45 each — **full, not PCA** — `expression`/`betas` 10).
A truncated hand pose, an extra field, or a bad `kind` fails validation. That is
the W0 acceptance criterion ("CI rejects malformed clips").

## Python (backend, dictionary build)
```sh
cd schemas/python
python -m pip install -e ".[dev]"

python -m pytest -q                         # contract tests
python -m asl_schemas.gen_json_schema       # regenerate JSON Schema
python -m asl_schemas.gen_json_schema --check   # CI: fail if stale
python -m asl_schemas.validate clip path/to/*.json   # validate clips (W6 uses this)
```
```python
from asl_schemas import SMPLXClip, SMPLXSequence, GlossSequence
clip = SMPLXClip.model_validate(data)       # raises on malformed input
```

## TypeScript (frontend / renderer)
```sh
cd schemas/ts && npm install && npm run build
```
```ts
import { SMPLXSequence, assertFrameDims, DIMS } from "@asl/schemas";
```
`assertFrameDims()` is a cheap runtime guard for the W5 player; TS can't enforce
tuple lengths, so the Python validator + JSON Schema are the real gate in CI.

## CI
`.github/workflows/schemas.yml` runs the pytest contract suite, the JSON-Schema
sync check, example-clip validation, and the TS typecheck/build on any change
under `schemas/`.
